TP N°4 de compression

Remarques: 

  • le sujet ne doit pas seulement être réalisé en TP: il nécessite que vous travailliez chez vous.
  • le travail réalisé doit être rendu dans la semaine qui suit le TP (sinon, note pour ce TP = 0).
  • les sujets ont souvent des dépendances: si vous ne réalisez/rendez pas les TPs des semaines précédentes, vous ne pourrez pas rendre les TPs des semaines suivantes.

Introduction:

Dans ce TP, nous étudierons les codages basés sur un dictionnaire.

1) Dictionnaire statique

On voudrait tester l'efficacité d'un codage avec un dictionnaire statique (à savoir, prendre un alphabet statique plus grand afin de stocker les motifs les plus courants). Les fonctions écrites devront permettre de choisir  le nombre de bits à utiliser pour étendre la taille de l'alphabet (exemple: si un alphabet fait 3 bits, 4 bits permet d'ajouter 8 symboles issus du dictionnaires, 5 bits 16 symboles, ...).
  1. Ecrire une fonction permettant de déterminer la fréquence les digrammes, afin de déterminer les plus fréquents pour la source considérée.
  2. Générer le code à taille fixe associé au dictionnaire statique (= symboles + digrammes les plus fréquents).
  3. Ecrire la fonction de codage binaire par dictionnaire statique. La fonction devra aussi retourner le taux de compression par rapport à un codage à taille fixe.
  4. Ecrire la fonction de décodage binaire. La fonction devra aussi retourner le taux d'expansion.
  5. Le bon fonctionnement sera vérifié sur l'ensemble des fichiers test. Les performances du codage seront mesurées sur l'ensemble des fichiers, comparées à l'entropie, et placé dans le fichier TP4.html.

2) Compression LZ78

On veut maintenant tester la méthode de compression LZ78, et étudier les performances de cet algorithme dans ses différentes variations. On devra aussi calculer le taux de compression par rapport à un codage à taille fixe.
  1. Ecrire une version de LZ78 en utilisant un dictionnaire sans taille maximale (à savoir le dictionnaire grandi indéfiniment). L'écriture des indices du dictionnaire devra utiliser le nombre de bits minimal possible au regard du nombre d'entrées déjà présentes dans le dictionnaire.
  2. Ecrire une première variation du LZ78 qui utilise une taille maximale de dictionnaire. Lorsque cette taille maximale est atteinte, on utilisera le dictionnaire comme un dictionnaire à taille fixe.
  3. Ecrire une seconde variation du LZ78 basée sur la précédente, qui comptabilise le nombre de fois qu'un code est utilisé lors des dernières entrées, et qui régulièrement (par exemple, toutes les 256 sorties de symboles), supprime du dictionnaire les codes moins fréquents (par exemple, les 16).
  4. Comparer les performances sur les fichiers de données de ces trois variations.

3) Ecriture dans un fichier

  1. Ajouter les fonctions d'écriture/lecture d'un codage avec dictionnaire à taille fixe dans un fichier.
  2. Ajouter les fonctions d'écriture/lecture d'un codage des différentes variations de LZ78 dans un fichier.

Remise de ce TP 

Mêmes conditions que le TP1 + TP2 + TP3. Le fichier devant contenir vos résultats et la déclaration devra s'appeler TP4.html.
